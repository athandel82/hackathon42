# Knowledge Base - Sample 1

## Prompt Used

The following prompt was used to generate this knowledge base:

> I would like to create an knowledge database based on the ARXML files stored in input/autosar/ARXML. This knowledge database should be done in a set of markdown files with specific structure one markdown file per components. Lets put this into output sample_1. Please document who this sample was generated.

## Overview

This knowledge base was generated from AUTOSAR ARXML files describing an Electronic Door Control (EDC) system demo. The system manages left and right door lock/open status, door commands, and LED indicators.

## Source Files

| File | Description |
|------|-------------|
| `input/autosar/ARXML/ArcCore_Types.arxml` | Platform base types and implementation data types (AUTOSAR R4.0) |
| `input/autosar/ARXML/EcuExtract.arxml` | ECU Extract with SW components, interfaces, composition, and communication (AUTOSAR R4.1.1) |

## Generation Details

## How This Was Generated

1. The ARXML files in `input/autosar/ARXML/` were parsed to identify AUTOSAR elements (components, interfaces, types, system configuration).
2. Each significant element was extracted into its own Markdown file with a consistent structure including: identification, description, ports, internal behavior, events, runnables, data access, and relationships.
3. Cross-references between components are preserved as AUTOSAR path references.
4. The knowledge base is intended to provide a human-readable, searchable reference for the AUTOSAR architecture defined in the source files.
